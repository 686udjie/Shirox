## Build & Run

The Xcode project is generated by **xcodegen** from `project.yml`. Regenerate after editing `project.yml`:

```bash
xcodegen generate
```

Build from the command line (scheme names use underscores):

```bash
# iOS (simulator)
xcodebuild -project yuhanime.xcodeproj -scheme yuhanime_iOS \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro' -configuration Debug build

# macOS
xcodebuild -project yuhanime.xcodeproj -scheme yuhanime_macOS \
  -destination 'platform=macOS' -configuration Debug build
```

Open in Xcode: `open yuhanime.xcodeproj`

Targets: **iOS 17+** and **macOS 14+** (native, not Catalyst). `Info.plist` must contain `CFBundleIdentifier`, `CFBundleName`, `CFBundleExecutable`, `CFBundlePackageType`, `CFBundleShortVersionString`, and `CFBundleVersion` — Xcode does not inject these automatically when a custom `INFOPLIST_FILE` is specified.

## Architecture

**MVVM + JSEngine module loader.** Source-specific scraping logic lives in JavaScript modules loaded at runtime via `JavaScriptCore`. The Swift app provides a generic UI and bridges (`fetchv2`, `console`, `btoa`/`atob`) so JS modules can make HTTP requests and parse responses.

### Data flow

```
ModuleListView → ModuleManager.addModule(jsonURL)
               → Downloads module JSON → stores ModuleDefinition
               → Downloads JS from scriptUrl → evaluates in JSContext

SearchView → SearchViewModel → JSEngine.search(keyword)
                              → calls JS searchResults(keyword)
                                        ↓ [SearchItem]
DetailView ← NavigationLink

DetailView → DetailViewModel → JSEngine.fetchDetails(url)   (JS extractDetails)
                             → JSEngine.fetchEpisodes(url)   (JS extractEpisodes)
                                        ↓ [EpisodeLink]
EpisodeRowView tap → DetailViewModel.loadStreams()
                   → JSEngine.fetchStreams(episodeUrl)        (JS extractStreamUrl)
                                        ↓ [StreamResult] (includes headers)
StreamPickerView (sheet) → user picks → PlayerView (fullScreenCover on iOS, sheet on macOS)
```

### JS Module Contract

Modules are defined by a JSON manifest (e.g., `animepahe.json`) with a `scriptUrl` pointing to a JS file. The JS file must export these async functions (all return JSON strings):

- `searchResults(keyword)` → `[{title, image, href}]`
- `extractDetails(url)` → `[{description, aliases, airdate}]`
- `extractEpisodes(url)` → `[{href, number}]`
- `extractStreamUrl(url)` → `{streams: [{title, streamUrl, headers}], subtitle: ""}`

JS modules can use `fetchv2(url, headers, method, body)` which is bridged to Swift's `URLSession`. Also available: `console.log/error`, `btoa/atob`, and HTML scraping utilities (`getElementsByTag`, `getAttribute`, `getInnerText`, `stripHtml`).

### Key services

| File | Responsibility |
|---|---|
| `Services/JSEngine.swift` | `@MainActor` singleton holding a `JSContext`. Sets up bridged functions (`fetchv2`, `console`, `btoa`/`atob`). Loads module scripts, calls JS functions, resolves Promises via `.then()`/`.catch()`. |
| `Services/JSEngine+Search.swift` | Calls `searchResults(keyword)` → parses JSON → `[SearchItem]` |
| `Services/JSEngine+Details.swift` | Calls `extractDetails(url)` and `extractEpisodes(url)` → parses JSON |
| `Services/JSEngine+Streams.swift` | Calls `extractStreamUrl(url)` → parses stream formats → `[StreamResult]` |
| `Services/ModuleManager.swift` | `@MainActor ObservableObject`. Manages module CRUD, persists to UserDefaults, loads active module into JSEngine on launch. |

### Non-obvious constraints

**AVPlayer header injection** — Stream modules may return custom headers (e.g., Kwik requires `Referer`/`Origin`). Headers are passed via `AVURLAsset` options, not `URLRequest`:
```swift
AVURLAsset(url: url, options: ["AVURLAssetHTTPHeaderFieldsKey": headers])
```

**JSContext Promise resolution** — JS module functions return Promises. Swift resolves them via `promise.invokeMethod("then", ...)` with `@convention(block)` callbacks. The `fetchv2` bridge creates Promises that resolve from `URLSession` completion handlers dispatched back to the main thread.

**Platform conditionals** — iOS-only APIs are wrapped in `#if os(iOS)`: `.navigationBarTitleDisplayMode()`, `.toolbarBackground(.hidden)`, `.fullScreenCover()` (macOS falls back to `.sheet`), `.statusBarHidden()`, and `.persistentSystemOverlays(.hidden)` in `PlayerView`.

**PlayerView custom controls** — On iOS, `VideoLayerView` wraps `AVPlayerViewController` with `showsPlaybackControls = false` so AVKit's native control bar is hidden. A custom overlay (dismiss button + title, gradient top bar) sits in the ZStack above it. The outer ZStack uses `.simultaneousGesture(TapGesture())` so tapping toggles the overlay visibility without blocking the dismiss `Button`.
